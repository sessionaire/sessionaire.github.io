Frequently Asked Questions

How do I install Sessionaire?

There are two ways to install Sessionaire:

  1. Firefox Add-ons (Recommended): Visit the Firefox Add-ons page and click "Add to Firefox".
  2. Download XPI: Download the .xpi file from the Releases page, then go to about:addons → ⚙️ → Install Add-on From File.
How does crash recovery work?

Sessionaire uses a heartbeat system to detect unexpected browser shutdowns:

  • The background script sends periodic "heartbeats" to track browser state
  • If Firefox crashes or is force-quit, the heartbeat stops unexpectedly
  • On next startup, Sessionaire detects the missing graceful shutdown
  • A recovery banner appears offering to restore your previous tabs
  • Click "Restore" to open all tabs in a new window, or "Dismiss" to skip

Note: Crash recovery can be enabled/disabled in Settings → General.

What is Auto Save and how does it work?

Auto Save automatically captures your current browsing state:

  • When enabled, Sessionaire saves a snapshot of all open tabs every time you make changes
  • The "Auto Save" session always appears at the top of your sessions list
  • It updates in real-time as you open, close, or navigate tabs
  • Uses debouncing (300ms delay) to avoid excessive saves during rapid changes
  • On Firefox startup, the previous Auto Save is renamed with a timestamp for reference

Enable/disable in Settings → General → Auto-Save Sessions.

Is my data private?

Yes! Sessionaire is 100% private:

  • Local storage only: All data is stored on your device using Firefox's built-in storage API
  • No external servers: Nothing is ever sent to external servers
  • No analytics: We don't collect any usage data or tracking
  • No ads: Completely ad-free experience
How do I export and import my sessions?

To export:

  1. Open Sessionaire full page view (click ⛶ expand button)
  2. Go to Settings → Backup & Restore
  3. Choose: Full Backup (sessions + containers + settings), Sessions Only, or Containers Only
  4. For individual sessions: click the menu (⋮) on any session → Export → choose format (JSON, XML, CSV, or Markdown)

To import:

  1. Go to Settings → Backup & Restore → Import Data
  2. Select your backup file (.json)
  3. Preview what will be imported
  4. Click "Confirm Import"
What are the keyboard shortcuts?
  • Alt+Shift+↓ - Open Sessionaire full page
  • Alt+Shift+↑ - Quick save current session (opens save dialog)

On Mac, use Ctrl+Shift instead of Alt+Shift.

You can customize shortcuts in Firefox: about:addons → ⚙️ → Manage Extension Shortcuts.

How do containers work with Sessionaire?

Sessionaire fully supports Firefox containers:

  • Container preservation: When saving sessions, each tab's container is saved
  • Container restoration: Tabs are restored in their original containers
  • Container management: Create, edit, and delete containers from the Containers tab
  • New tab selector: When enabled, new tabs show a container picker (Settings → General)
  • Visual indicators: Container colors are shown on tabs in session views
What's the difference between popup and full page view?

Popup (click toolbar icon):

  • Quick access to sessions and current tabs
  • Save and restore sessions
  • Compact 600×800px interface

Full page (click ⛶ or use keyboard shortcut):

  • All popup features plus:
  • Grid/List view toggle for sessions
  • Container management
  • Settings and preferences
  • Backup & Restore
  • Session editing with tab management
Can I edit a saved session?

Yes! In full page view, you can fully edit any session:

  • Click the menu (⋮) on any session → Edit
  • Rename the session
  • Change the session color
  • Add or remove windows
  • Add, edit, or delete individual tabs
  • Reorder tabs within windows

Troubleshooting

Extension doesn't load or icon is missing

The extension icon doesn't appear in the toolbar or shows as disabled.

  • • Verify Firefox version is 57 or later (type about:support to check)
  • • Right-click the toolbar → Customize → drag Sessionaire icon to toolbar
  • • Check about:addons to ensure the extension is enabled
  • • Try disabling and re-enabling the extension
  • • Reinstall the extension from Firefox Add-ons

Sessions not saving

Manual or auto-save sessions are not being created.

  • • Ensure you have at least one valid tab open (not about: or moz-extension: pages)
  • • Check if the "Save Session" button is disabled (hover for tooltip)
  • • Open Browser Console (Ctrl+Shift+J) and look for error messages
  • • Check storage quota: about:preferences#privacy → Cookies and Site Data
  • • Try clearing extension storage and reimporting your backup

Auto-save not working

The Auto Save session is not updating when tabs change.

  • • Check Settings → General → "Auto-Save Sessions" is enabled
  • • Auto-save uses 300ms debouncing - wait a moment after making changes
  • • Look for "Auto-save triggered" in Browser Console for confirmation
  • • Restart Firefox to reinitialize the background script

Crash recovery not appearing

The recovery banner doesn't show after a crash.

  • • Ensure Settings → General → "Crash Recovery" is enabled
  • • Recovery only triggers for unexpected shutdowns (crashes, force-quit)
  • • Normal browser close is detected as graceful shutdown
  • • Check Browser Console for "Crash recovery" messages
  • • The recovery data expires after a period of time

Tabs not restoring in correct containers

Restored tabs open in default container instead of their saved container.

  • • Ensure the container still exists (wasn't deleted after saving)
  • • Container names must match exactly - renamed containers won't match
  • • Check if Firefox Multi-Account Containers extension is installed and enabled
  • • Some URLs (about:, moz-extension:) cannot be opened in containers

Tab highlighting not updating

Open tabs in sessions don't show as highlighted, or highlighting is delayed.

  • • Tab highlighting updates in real-time but may have slight delay
  • • Highlighting checks URL match - tabs with different URLs won't match
  • • Try switching to another view and back to refresh
  • • Check Browser Console for any JavaScript errors

New tab container selector not showing

Opening a new tab doesn't show the container picker.

  • • Enable in Settings → General → "New Tab Container Selector"
  • • The selector only appears for new tabs, not when navigating
  • • Check if another extension is overriding the new tab page
  • • Verify the extension has the "newtab" permission in about:addons

Performance issues or slow loading

The extension is slow or Firefox lags when using Sessionaire.

  • • Delete old sessions you no longer need (recommended: keep under 50)
  • • Use the popup view for quick actions instead of full page
  • • Disable auto-save if you don't need real-time session tracking
  • • Export and clear sessions periodically, then reimport if needed
  • • Check Browser Console for repeated error messages
